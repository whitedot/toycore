# 보안 체크리스트

Toycore의 기능을 추가하거나 수정할 때 확인할 최소 보안 체크리스트입니다.

## 공통

- 외부 입력을 직접 신뢰하지 않았는가
- enum 값은 허용 목록으로 검증했는가
- 숫자 ID는 정수 변환과 범위 확인을 했는가
- 출력값은 컨텍스트에 맞게 escape했는가
- 오류 화면에 민감 정보가 노출되지 않는가
- 로그에 비밀번호, 토큰, 개인정보 원문이 남지 않는가

## DB

- 모든 값 바인딩에 prepared statement를 사용했는가
- 테이블명, 컬럼명, 정렬 방향은 허용 목록으로 제한했는가
- SQL 오류를 운영 화면에 그대로 출력하지 않는가
- 설치/업데이트 SQL은 재실행 위험을 검토했는가

## 인증과 세션

- 비밀번호는 `password_hash()`로 저장하는가
- 비밀번호 검증은 `password_verify()`를 사용하는가
- 로그인 성공 후 `session_regenerate_id(true)`를 실행하는가
- 세션 쿠키에 `HttpOnly`, `Secure`, `SameSite`를 적용하는가
- 장기 로그인이나 재설정 토큰은 hash만 저장하는가
- 로그아웃 시 관련 세션/토큰을 폐기하는가

## CSRF

- 모든 상태 변경 요청에 CSRF 검증이 있는가
- 로그인 후 CSRF 토큰을 갱신하는가
- 관리자 화면의 POST 요청도 CSRF 검증을 하는가

## 권한

- 로그인 확인과 권한 확인을 분리했는가
- 화면 버튼 숨김에만 의존하지 않고 서버에서 권한을 검증하는가
- 관리자 action 파일 진입 시 권한을 먼저 확인하는가

## 모듈과 include

- 모듈 키는 허용된 형식으로 검증했는가
- action 파일 경로는 모듈 디렉터리 내부 상대 경로로 제한했는가
- 활성화되지 않은 모듈 action 파일을 include하지 않는가
- 설정값을 검증 없이 파일 경로로 조립하지 않는가

## 관리자

- 관리자 화면은 기본 `noindex`인가
- 설정 변경, 모듈 변경, 회원 상태 변경은 감사 로그에 남는가
- 개인정보 조회/처리 작업은 별도 로그를 남기는가
- 관리자 화면에도 XSS 방어가 적용되는가

## 개인정보

- 필수 정보와 선택 정보를 분리했는가
- 동의가 필요한 항목은 버전과 시점을 기록하는가
- 삭제/익명화 정책을 확인했는가
- 개인정보 요청 이력은 계정 삭제 후에도 보존 가능한가

## 배포 경로

- 루트 `index.php`만 공개 진입점으로 사용하는가
- `config/`, `storage/`, `database/`, `core/`, `modules/`, `docs/`, `examples/` 직접 접근을 운영 환경에서 차단했는가
- `AGENTS.md`, `README.md` 같은 루트 보조 파일이 웹에서 열리지 않는가

## 파일 업로드

- 업로드 파일 크기를 제한했는가
- 실행 가능한 확장자를 차단했는가
- 원본 파일명으로 저장하지 않는가
- 가능하면 웹에서 직접 접근할 수 없는 위치에 저장하는가
- 다운로드는 직접 경로 노출 대신 action 파일을 통하는가

## 캐시

- 로그인 상태나 권한에 따라 달라지는 HTML을 캐시하지 않는가
- CSRF 토큰이 포함된 화면을 캐시하지 않는가
- 개인정보가 포함된 화면을 캐시하지 않는가
- locale처럼 출력에 영향을 주는 값이 캐시 키에 반영되는가
