# 기본 코어 우선순위

이 문서는 `member` 모듈을 제외한 Toycore 기본 코어의 우선순위를 정리합니다.

회원가입, 로그인, 비밀번호 재설정, 회원 프로필, 회원 동의 처리는 `member` 모듈 책임입니다. 기본 코어는 그런 모듈들이 안전하게 설치되고 실행될 수 있는 기반만 제공합니다.

단, 실제 기본 설치에서는 `member`와 `admin`을 필수 기본 모듈로 함께 설치합니다. 이 문서에서 `member`를 제외한다는 것은 코어 책임을 검토할 때 인증 도메인 구현을 분리해서 본다는 뜻입니다.

## 기본 원칙

```text
코어는 실행 기반
member는 인증 도메인
admin은 관리 화면
각 모듈은 자기 기능
```

코어는 다음을 직접 구현하지 않습니다.

- 회원가입
- 로그인
- 로그아웃
- 비밀번호 재설정
- 회원 프로필
- 게시판/페이지/상품 콘텐츠
- 콘텐츠별 SEO
- 콘텐츠 다국어

## 추가 우선순위

### 1. 설치/초기화

가장 먼저 필요합니다.

이유:

- 저가형 웹호스팅에서는 CLI보다 웹 설치가 중요함
- DB 접속과 기본 테이블 생성이 되어야 이후 기능 검증 가능
- 최초 관리자 계정과 기본 모듈 상태를 정해야 함

문서:

- [설치 및 초기화 계획](install-plan.md)

### 2. 업데이트/스키마 버전 관리

설치 다음으로 필요합니다.

이유:

- 모듈 구조는 설치뿐 아니라 업데이트 기준이 있어야 유지 가능
- 프레임워크 migration은 피하더라도 SQL 버전 기록은 필요
- 운영 중 DB 구조 변경을 추적해야 함

문서:

- [업데이트 및 스키마 버전 계획](update-plan.md)

### 3. 운영 모드와 에러 처리

초기부터 기준을 잡아야 합니다.

이유:

- 절차형 PHP는 운영 중 에러 노출 위험이 큼
- 설치 전/설치 후/점검 모드/운영 모드 처리가 필요
- 404, 500, DB 연결 실패 화면 기준이 필요

문서:

- [운영 모드 및 에러 처리 계획](runtime-ops-plan.md)

### 4. 감사 로그

관리자 화면보다 먼저 설계 기준이 필요합니다.

이유:

- 설정 변경, 모듈 활성화, 관리자 작업은 추적되어야 함
- GDPR과 보안 운영에서 관리자 작업 이력이 중요함
- 코어는 로그 helper와 테이블을 제공하고, 각 모듈은 이벤트를 기록할 수 있어야 함

문서:

- [감사 로그 계획](audit-log-plan.md)

### 5. 보안 체크리스트

구현이 시작되기 전에 있어야 합니다.

이유:

- 기능 추가 시 반복적으로 확인할 기준 필요
- AI가 생성한 코드도 동일한 보안 기준으로 검토해야 함
- 절차형 구조에서는 실수 방지 체크리스트가 특히 중요함

문서:

- [보안 체크리스트](security-checklist.md)
